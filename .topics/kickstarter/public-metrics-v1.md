# Public Metrics v1

## Purpose

The goal of public metrics is not surveillance.

The goal is operational visibility:
allowing observers to understand how the project evolves over time.

---

## Metrics Being Explored

Potential public metrics include:
- repository activity
- milestone progression
- continuity exports
- trace generation
- tooling progress
- development focus shifts
- funding state
- token usage
- recoverability experiments

---

## Funding Visibility

Funding visibility may include:
- salary withdrawals
- infrastructure costs
- tooling costs
- token/API expenses
- project allocations
- bonus allocations

The intention is explicit project continuity, not personal financial surveillance.

---

## Redacted / Omitted Data

The system intentionally supports:
- redacted entries
- omitted information
- selective disclosure
- bounded transparency

Not all continuity must be public to remain structurally honest.

---

## Why Metrics Matter

Metrics help:
- reduce ambiguity
- preserve historical context
- improve recoverability
- enable feedback loops
- support public trust

---

## Important Constraint

Metrics should never replace human judgment.

Signals require interpretation, context, and ongoing feedback.
